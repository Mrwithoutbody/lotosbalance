4. Co już istnieje, a co wymaga budowy

Frontend działa. Brakuje backendu, płatności, marketplace.

Zbudowane (Q1 2026)

  • Frontend: React + TypeScript + Vite
  • Interfejs 3D kwiatu lotosu: Three.js + R3F + drei. Proceduralna geometria, nawigacja wielopoziomowa.
  • 54 mikro-praktyki w 6 filarach z systemem odblokowania
  • Śledzenie postępu (localStorage)
  • Responsywny design — testowane na iPhone SE. Ciemny i jasny motyw.

Demo: app.lotosbalance.pl

Do zbudowania

  • Backend: konta, profile, sync postępu, streaki (Supabase / PostgreSQL)
  • System afirmacji: baza tematyczna, engine doboru, push (FCM)
  • Platforma twórców: upload, panel, linki sprzedażowe, tracking prowizji, partner program
  • lotosPayments: warstwa płatności na protokole x402 (Coinbase + Cloudflare + Linux Foundation), settlement na Base L2, BLIK/przelew
  • Marketplace specjalistów: katalog, profile, booking, kalendarz, panel specjalisty
  • AI-asystent: rekomendacje praktyk i specjalistów (LLM API)
  • Społeczność: grupy tematyczne, prosty feed

Cel finansowania: zamknąć backend + platformę twórców + lotosPayments + marketplace + afirmacje tak, by produkt nadawał się do publicznego launchu B2C.
